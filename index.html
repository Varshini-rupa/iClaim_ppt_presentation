Slide 2 — The Challenge
THE CHALLENGE
Business Problem & Industry Context
"Meet Ramesh. He is a 45-year-old schoolteacher cleared for hospital discharge at 9:00 AM. It is now 9:00 PM, and he is still waiting. Why? Because the hospital desk and the insurance underwriters are locked in a slow, manual cycle of verification — cross-checking doctor IDs, manually reading medical bills, and scanning for duplicates. Ramesh's delay is the human cost of manual claims processing."
🐢
Slow Manual Review
Processing takes days to weeks per claim. Reviewers manually cross-check hundreds of documents per day.
⚖️
Inconsistent Rules
Manual policy checks vary reviewer to reviewer, causing reviewer fatigue and inconsistent outcomes.
🚨
Fraud Exposure
Undetected duplicate bills and unverified doctor credentials drive major financial losses every year.

Slide 5 — The Solution
THE SOLUTION
Three Core AI Pillars powering iClaim
Pillar 1 — Document Intelligence
OCR Extraction Layer
⚡ Tesseract OCR + PyMuPDF
Automatically extracts structured text from uploaded hospital bills, prescriptions, and discharge summaries. Handles both native-text PDFs and scanned image documents via fallback Tesseract pipeline.
Pillar 2 — Policy Intelligence
Policy RAG Layer
⚡ spaCy Vector Semantics + SQLite VectorStore
Performs semantic cosine similarity search across 17 insurance policy PDFs. Answers 4 structured questions per claim: coverage, exclusions, waiting periods, and pre-authorization requirements.
Pillar 3 — Fraud Intelligence
ML Fraud Ensemble Layer
⚡ Random Forest + Gradient Boosting + Isolation Forest
Analyzes 33 extracted claim features. Composite ensemble scores fraud probability 0.0–1.0. Combines with 6-rule validators for explainable, auditable risk output.
Human-in-the-Loop
Reviewer Governance
⚡ Explainable Recommendations + HITL Override
Medical reviewer confirms, overrides, and finalizes every assessment. AI assists — reviewers decide. Full audit trail on every action taken.
🧮 Risk Engine
Composite 0–100 risk score blending financial, fraud, and coverage components.
🛡️ 6 Rule Validators
Checks dates, network hospitals, doctor licenses, bills, policy rules, duplicates.
🌐 Bilingual Chatbot
Groq LLaMA-3 powered assistant in Telugu & English with guardrails.

Slide 10 — Technical Highlights
TECHNICAL HIGHLIGHTS
Engineering Challenges Solved
🛡️
AI Chatbot Guardrails
Implemented prompt injection blockers, Telugu/English toxicity filters, and off-topic keyword checks in FastAPI to prevent AI instruction bypasses. Branding output post-processor sanitizes all LLM responses for strict iClaim Insurance compliance.
🗃️
SQLite VectorStore
ChromaDB replaced with a lightweight SQLite vector_store table. spaCy en_core_web_sm vectors serialized as JSON strings. Cosine similarity computed in-process — no external network dependency. Drastically reduces deployment footprint.
📐
Collapsible Layout States
Sidebar state memory using localStorage across all reviewer, admin, and customer dashboards. Prevents visual clutter on dense claim queues. toggleSidebar() preserves state on page reload.
🌐
Bilingual i18n Engine
data-i18n attributes on all client-facing elements. i18n.js dynamically swaps text on language toggle. Regex rules /POL\d+/ and /CLM-\d{4}-\d+/ protect policy and claim IDs from translation corruption.
🎤
Voice Input (Web Speech API)
Native SpeechRecognition initialized per language (en-US / te-IN). Animated pulsing microphone button triggers recognition. Auto-populates chat input and submits to POST /chat. Offline findAnswer() fallback handles API failures.
🔄
HITL Override & Fallback Chain
POST /assessment/override/{claim_id} recalculates composite risk with reviewer-supplied rule overrides. LLM fallback: Groq → HuggingFace → Mock Engine. Web search fallback: DuckDuckGo → Bing Scraper. SMTP fallback: console-printed OTP.
Slide 11 — Testing & QA
TESTING & QA
Validation Framework — Robust Verification Pipeline
🧪
Automated Test Suites
FastAPI TestClient integrations verifying claim status transitions, schema compliance, and guardrail enforcement. Includes test_plans.py for dynamic policy plan creation endpoint validation.
🤖
Mock Fallback Testing
Robust local mocked responders within llm_client.py ensuring help questions (login flows, password resets) work even when Groq and HuggingFace API endpoints are offline or rate-limited.
📊
Coverage Verification
Comprehensive test coverage across claim database transitions — submitted → ai_assessed → under_review → approved / rejected / escalated. Status machine edge cases validated in full claim lifecycle.
🔁
CI/CD Pipeline Testing
Jenkins Jenkinsfile.ci runs pytest on every Bitbucket push. flake8 linting enforced. Pipeline tests include fraud model loading, DB seed integrity, and policy ingestion into the SQLite VectorStore.
🕵️
Fraud Model Validation
Ensemble model (RF + GB + IsoForest) trained on 2,000+ synthetic records with 80/20 legitimate/fraud split. 5-fold cross-validation. Precision, recall, and F1 score tracked per training run.
🌍
Bilingual Guardrail QA
Telugu and English toxicity filter unit tests. Regex protection rules for POL\d+ and CLM-\d{4}-\d+ verified post-translation. Brand enforcement checks applied to all LLM output strings.
